Setup & Help

Get rolling with AlForks.

A local app for browsing, mapping, and analysing your GPS rides — running entirely on your own computer. Follow the waypoints below; the whole setup takes about five minutes.

Runs at localhost:5000 Version {{ version }} Windows · Python 3.10+
  1. 01
    Waypoint 01

    Launch AlForks

    Double-click start.bat. The first run quietly sets up a small Python environment (about a minute); after that it’s instant. Your browser opens automatically to http://localhost:5000.

    • A black command window stays open while you use AlForks — that window is the app.
    • Close the window to stop it. Run start.bat again any time to start back up.
    First time on Windows? If you see “Windows protected your PC”, click More info → Run anyway. If a Firewall prompt appears, Allow it (or Cancel — localhost works either way). If it says Python is missing, install Python 3.10+ from python.org and tick “Add Python to PATH” during install.
  2. 02
    Waypoint 02 · the main step

    Connect your Strava

    This is what pulls your own rides into AlForks — Strava provides the GPS tracks themselves (Garmin, in step 5, is optional and only adds heart-rate on top). You create a small personal Strava “API application” once — about two minutes — then connect it.

    1. Open strava.com/settings/api and create an application with exactly these values:

    Not connecting? Double-check the Client ID / Secret were pasted cleanly and that the Callback Domain is exactly localhost (no http://, no port).
  3. 03
    Waypoint 03

    Choose how far back to import

    Pick a start date so your first sync only pulls rides from then onward — then sync, right here. Leave the date blank to import your entire history.

    Connect Strava (step 2) first. When the sync finishes, your rides appear throughout the app.

    Big Strava history? Importing years of rides at once can take several minutes (and very large libraries may time out). Setting a start date keeps that first sync quick.

Reference

Keeping up to date

When you receive an alforks-update-*.zip:

  • Close AlForks first (close the command window).
  • Drop the zip into your AlForks folder and double-click update.bat.
  • Restart with start.bat. The first launch after a big update may rebuild its cache for up to a minute — that’s normal.

Updates only replace program files — your rides, settings, and login are untouched.

Your data & privacy

  • Everything lives in your AlForks folder: your rides, settings, and Strava login.
  • Nothing leaves your computer except your own requests to Strava and (if enabled) Mapbox.
  • Don’t zip up and re-share your whole folder — it contains your Strava login.

Troubleshooting

  • Browser says “can’t connect”: give it a few seconds after launch, then refresh — the app may still be starting.
  • Still nothing: make sure only one copy of AlForks is running (they all share localhost:5000).
  • Sync seems stuck: a first sync of a large history just takes a while — set an “Import rides from” date to shrink it.

Good to know

  • Tick “Run Strava sync when AlForks starts” on the Sync tab to auto-import new rides each launch.
  • Your activity types and regions come pre-loaded as starting points — edit or delete them freely.
  • This guide lives at /guide — reachable any time from the Setup page.
